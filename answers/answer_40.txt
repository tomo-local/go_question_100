```go
package main

import "fmt"

type Person struct {
	Age int
}

// ポインタレシーバ (*Person)
// メソッド内でレシーバのフィールド値を変更する必要がある場合に使用する
// pは元のPersonインスタンスへのポインタなので、*pで値にアクセスして変更すると、
// 呼び出し元のインスタンス自体が変更される
func (p *Person) celebrate() {
	p.Age++ // (*p).Age++ と書くのと同じ（Goが自動で解釈してくれる）
}

// 値レシーバ (Person)
// レシーバのフィールド値を変更する必要がない場合（読み取り専用）に使用する
// pはPersonインスタンスのコピーなので、この中でp.Ageを変更しても呼び出し元には影響しない
func (p Person) getAge() int {
	return p.Age
}

func main() {
	// ポインタではなく、Person構造体の値としてインスタンスを作成
	person := Person{Age: 20}
	
	fmt.Printf("初期年齢: %d\n", person.getAge())

	// celebrate() を呼び出す
	// personはポインタではないが、Goが自動的に `(&person).celebrate()` と解釈してくれる
	person.celebrate()

	fmt.Printf("celebrate() 後の年齢: %d\n", person.getAge())
}
```

### 解説

- **ポインタレシーバ (`*T`)**:
  - **いつ使うか**: メソッド内でレシーバ（構造体など）のフィールドの値を**変更したい**場合。
  - **なぜ**: メソッドはレシーバのポインタ（アドレス）を受け取るため、元のインスタンスを直接変更できます。また、レシーバが大きな構造体の場合、コピーするコストを避けられるというパフォーマンス上の利点もあります。

- **値レシーバ (`T`)**:
  - **いつ使うか**: メソッドがレシーバのフィールドを**読み取るだけ**で、変更しない場合。
  - **なぜ**: メソッドはレシーバのコピーを受け取るため、元のインスタンスが意図せず変更される心配がありません。

- **慣習**:
  - ある型に対して、一つでもポインタレシーバのメソッドがあるなら、他のメソッドも（たとえフィールドを変更しなくても）ポインタレシーバで統一することが多いです。これにより、レシーバの型が混在する混乱を避けられます。
