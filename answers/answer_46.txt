```go
package main

import (
	"fmt"
	"strings"
)

func main() {
	// `interface{}` 型の変数に string を代入
	var i interface{} = "hello"

	// --- 方法1: value, ok 形式 (推奨) ---
	// この形式は、アサーションが失敗してもパニックにならない
	s, ok := i.(string)
	if ok {
		fmt.Println("アサーション成功 (方法1):", strings.ToUpper(s))
	} else {
		fmt.Println("アサーション失敗: iはstringではありません")
	}

	// --- 方法2: valueのみ形式 ---
	// この形式は、アサーションが失敗するとパニックを引き起こす
	s2 := i.(string)
	fmt.Println("アサーション成功 (方法2):", s2)


	// --- 失敗する例 ---
	var j interface{} = 123
	
	// value, ok 形式なら、okがfalseになるだけ
	_, ok = j.(string)
	if !ok {
		fmt.Println("j.(string) へのアサーションは失敗しました (安全)")
	}

	// valueのみ形式でアサーションすると、ここでパニックが発生するためコメントアウト
	// s3 := j.(string)
}
```

### 解説
- **型アサーション (Type Assertion)** は、インターフェース型の変数が、実際にその中に格納されている具体的な型（動的型）が何であるかを明らかにし、その型の値としてアクセスするための操作です。
- `i.(T)` という構文で、インターフェース `i` の値が型 `T` であると表明（assert）します。
- `value, ok := i.(T)`:
  - **最も安全で推奨される形式**です。
  - アサーションが成功すれば、`value` には `T` 型の値が、`ok` には `true` が入ります。
  - 失敗すれば、`value` には `T` 型のゼロ値が、`ok` には `false` が入ります。パニックは起きません。
- `value := i.(T)`:
  - アサーションが成功すれば `value` に値が入りますが、**失敗するとパニックが発生**してプログラムがクラッシュします。
  - インターフェースの値がその型であることが絶対に確実な場合にのみ使用すべきです。
